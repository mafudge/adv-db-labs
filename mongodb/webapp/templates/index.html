Mongo Db Example

This example demonstrates one of the advantages of a document (or any noSQL database for that matter) - it does not need to validate a schema.

For example we have two versions of a feedback form which collects data from the user and appends it to the feedback collection of the demo database

If we did this in a relational database we would need to update the business logic and data layers as well. We only need to update the presentation layer here!

The disadvantage of this implementation is that all data logic such as validation and constraint checking must now be implemented in the business logic layer.

  • Version 2 of the form. (schema changed, but writes to the same collection)
